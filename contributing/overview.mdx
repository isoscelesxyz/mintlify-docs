We welcome contributions to Aerodrome documentation! Whether you're fixing typos, clarifying concepts, or writing entirely new guides, your help makes Aerodrome more accessible to everyone.

## Ways to Contribute

**Fix and improve existing content:**

- Fix typos, broken links, and formatting issues
- Clarify confusing explanations
- Add missing examples or context
- Update outdated information

**Create new content:**

- Write missing documentation (see [Docs Roadmap](/contributing/roadmap))
- Add troubleshooting sections
- Contribute FAQ questions
- Create integration examples

**Review and feedback:**

- Suggest improvements via GitHub issues
- Review pull requests from other contributors
- Test documentation accuracy by following guides

## Before You Start

1. **Read the [Style Guide](/contributing/style-guide)** to understand our documentation philosophy, voice, and standards
2. **Check the [Docs Roadmap](/contributing/roadmap)** for areas where we need help
3. **Review existing pages** to match the tone and structure

## Documentation Philosophy

Aerodrome documentation is **task-oriented, not marketing**. Every page should enable the reader to complete at least one concrete job.

- **Be calm and technical**: No hype, no speculation
- **Be direct and precise**: Use second person ("you"), active voice, short sentences
- **Be risk-aware**: Always mention relevant risks appropriately
- **Be accurate**: Never guess data - link to canonical sources

See the full philosophy in our [Style Guide](/contributing/style-guide).

## How to Submit Changes

### Quick Edits (Typos, Small Fixes)

1. Find the "Edit this page" link at the bottom of any docs page
2. Make your changes directly on GitHub
3. Submit a pull request with a clear description
4. Our team will review and merge

### Larger Contributions (New Pages, Major Updates)

1. **Check first**: Comment on a [GitHub issue](https://github.com/aerodrome-finance/docs/issues) or create a new one to discuss your proposed changes
2. **Fork the repository**: Fork [aerodrome-finance/docs](https://github.com/aerodrome-finance/docs)
3. **Create a branch**: `git checkout -b docs/your-feature-name`
4. **Make your changes**: Follow the style guide and match existing page structures
5. **Test locally**: Run the docs locally to verify formatting and links
6. **Submit a PR**: Include a clear description of what you changed and why
7. **Respond to feedback**: Our team will review and may request changes

## What Happens After You Contribute?

1. **Review**: A docs maintainer will review your contribution
2. **Feedback**: You may receive suggestions for improvements
3. **Merge**: Once approved, your changes will be merged to `main`
4. **Deploy**: Changes to `main` auto-deploy to production immediately
5. **Credit**: Your contribution will be visible in the Git history

## Recognition

All contributors are listed in the repository's Git history. Significant contributors may be highlighted in release notes and community updates.
